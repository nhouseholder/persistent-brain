# Handoff — 2026-04-26 — unified-brain v0.6.1 (Auto-Calibration + Token Estimation)

## What we did

Completed two remaining items from the Kahneman v3.0 handoff:

1. **Auto-calibration pipeline** (`04d298c`)
   - `scripts/auto-calibrate.py` — checks `reasoning_state.json` at session end
   - Calibrates DELIBERATE/SLOW tasks automatically; skips FAST
   - Wired into `hooks/session-end.sh` between timeline close and stats output
   - Infers `would_fast_have_sufficed`: pulls ≤1 → "likely yes"; pulls > budget → "likely no"

2. **Token estimation heuristic** (`e3d04f6`)
   - `router/brain_router.py` — after every tool call, estimates tokens from response JSON size
   - Heuristic: `len(json.dumps(result)) // 4` ≈ token count (1 token ≈ 4 chars)
   - Passed to `reasoning_tracker.record_pull(tool_name, estimated_tokens)`
   - Makes token budget self-enforcing with zero agent action
   - Large responses (brain_diagram, codebase_search) naturally consume more budget

## Why we did it

The v0.6.0 handoff identified both as last unimplemented pieces of Kahneman v3.0:
- Calibration was dead code because agents never remembered to call `brain_calibrate` manually
- Token budget was unenforceable because `token_estimate` param was never passed manually

## How we did it

- Auto-calibrate: Python script imports `reasoning_tracker` and `brain_router`, ends pending tasks, calls `handle_brain_calibrate()` for deliberate/slow modes
- Token estimation: Simple string-length heuristic in `handle_request()` tools/call branch. Zero dependencies, conservative (slightly overestimates), no tokenizer needed.
